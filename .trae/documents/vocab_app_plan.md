# 英语单词背诵软件 - 项目计划

## 一、项目目的

本项目旨在开发一款适合学生背诵英语单词的软件，核心目标包括：

1. **内置单词库**：提供高考要求的默认单词库，支持用户自定义添加单词
2. **智能复习**：基于记忆曲线算法，科学安排复习计划，避免复习任务超标
3. **双向背诵**：支持中译英（输入英文）和英译中（选择中文释义）两种模式
4. **进度追踪**：记录每次背诵情况，评估单词掌握程度
5. **防重复机制**：避免用户录入已存在的单词

| 字段 | 含义 | 类型 |


| 字段 | 含义 | 类型 |
|------|------|------|
| `p` | 词性 | 字符串（如 n., v., adj., adv.） |
| `c` | 中文释义数组 | 字符串数组 |

## 三、记忆曲线算法

### 3.1 算法原理

采用简化的艾宾浩斯记忆曲线模型，结合可调节参数评估单词掌握程度：

**核心公式**：
```
nextReviewDays = baseInterval * (easeFactor ^ reviewCount)
```

### 3.2 参数定义

| 参数 | 默认值 | 含义 |
|------|--------|------|
| `baseInterval` | 1 | 基础间隔天数（第一次背诵后的复习间隔） |
| `easeFactor` | 1.5 | 难度系数，决定间隔增长速度 |
| `minInterval` | 1 | 最小复习间隔（防止过短间隔） |
| `maxInterval` | 100 | 最大复习间隔（防止过长间隔） |
| `dailyReviewCap` | 用户设定 | 每日最大复习单词数 |

### 3.3 背诵情况对难度系数的影响

| 背诵情况 | 难度系数变化 |
|----------|-------------|
| 对 | easeFactor += 0.1 |
| 错 | easeFactor = max(1.2, easeFactor - 0.2) |
| 不熟 | easeFactor = max(1.3, easeFactor - 0.1) |

### 3.4 复习日期计算流程

1. 根据单词当前的复习次数 `reviewCount` 计算下次复习天数
2. 从最后一次背诵日期 `lastReviewDate` 加上计算的天数
3. 得到 `nextReviewDate`

**示例**：
- 第1次背诵（学习）→ 第2天复习（间隔1天）
- 第2次背诵（复习）正确 → 第1+1×1.5=2.5天 → 第3天复习
- 第3次背诵（复习）正确 → 第1×1.5²=2.25天 → 约3天后复习

### 3.5 今日任务生成逻辑

1. **背诵单词**：随机抽取从未背诵过的单词（所有r字段为空），数量=用户设定的背诵数量
2. **复习单词**：
   - 找出所有 `nextReviewDate ≤ 今天` 的单词
   - 按 `nextReviewDate` 升序排序（越接近今天越优先）
   - 截取前N个（N=用户设定的复习数量）
3. **防超量机制**：如果复习候选单词超过用户设定的数量，按优先级筛选

## 四、功能规格

### 4.1 核心功能

#### 4.1.1 单词库管理

| 功能 | 说明 |
|------|------|
| 默认词库 | 内置高考英语核心词汇（约3500词） |
| 导入单词 | 支持用户添加单个单词及其释义 |
| 防重复 | 添加前检查词库，避免重复录入 |
| 词库查询 | 根据英文单词查找释义 |

#### 4.1.2 背诵功能

| 功能 | 说明 |
|------|------|
| 中译英 | 显示中文释义，用户输入英文单词 |
| 英译中 | 显示英文单词，从6个随机选项中选择正确中文释义 |
| 答案检测 | 中译英：大小写不敏感匹配；英译中：选项匹配 |
| 背诵标记 | 支持"对"、"错"、"不熟"三种标记，快捷键操作 |
| 切换模式 | 背诵过程中可切换中译英/英译中模式 |

#### 4.1.3 复习计划

| 功能 | 说明 |
|------|------|
| 今日任务 | 用户设定背诵数量和复习数量，系统生成任务 |
| 自动复习 | 根据记忆曲线算法自动计算应复习单词 |
| 进度追踪 | 记录每次背诵结果，更新单词状态 |

### 4.2 快捷键

| 按键 | 功能 |
|------|------|
| Enter | 提交答案（中译英）/切换下一个（英译中） |
| 0 | 标记为"对" |
| 8 | 标记为"不熟" |
| 9 | 标记为"错" |

## 八、潜在风险与处理

### 8.1 英译中选项不足问题

**问题**：当词库中单词较少时，可能无法生成6个不同的中文释义选项。

**解决方案**：
- 如果词库中不同释义少于6个，减少选项数量
- 添加占位选项（如"其他"、"以上都不是"）

### 8.2 复习任务超标问题

**问题**：根据记忆曲线，可能某天需要复习的单词数量超过用户设定的上限。

**解决方案**：
- 按复习优先级排序（过期单词 > 今日到期单词）
- 只取前N个单词作为今日复习任务
- 剩余单词自动顺延到次日

### 8.3 答案匹配精度问题

**问题**：中译英模式下，用户输入可能包含大小写、空格等差异。

**解决方案**：
- 去除首尾空格
- 转换为小写后比较
- 支持部分匹配（如输入"appl"可提示"apple"）

### 8.4 数据丢失风险

**问题**：localStorage 数据可能被清除。

**解决方案**：
- 定期导出词库数据（JSON格式）
- 提供导入功能恢复数据

## 九、逻辑检查与改进

### 9.1 逻辑矛盾检查

**检查点1**：单词背诵记录只有10次，但记忆曲线可能需要更多次复习。
- **结论**：10次记录足够覆盖大部分单词的有效记忆周期。超过10次后，单词已基本掌握，可停止记录详细信息。

**检查点2**：英译中选项从"今日任务"中其他单词的释义选取，可能出现正确答案重复。
- **结论**：需要去重处理，确保6个选项中正确答案唯一。

**检查点3**：复习优先级排序是否合理。
- **结论**：按 `nextReviewDate` 升序排序，过期单词优先，符合记忆曲线原理。

### 9.2 改进建议

1. **添加导出功能**：允许用户导出词库数据作为备份
2. **添加统计图表**：展示背诵进度、正确率趋势等数据可视化
3. **添加学习提醒**：根据记忆曲线提醒用户复习时间
4. **支持批量导入**：允许用户从文件批量导入单词
5. **添加单词分类**：支持按词性、难度等维度分类管理
